A5 - Getting Meshy - Playing with Meshes in Rhino

Prompt Modify the mesh of two STLs from Thingiverse to produce a single, printable STL.

Looking for lamps? I've combined all lamp reports into A6.

Inspiration I love Pokemon! Luckily, so do many Thingiverse users. I wanted to do something with a Pokemon mesh. In class we did a demo of placing a character onto a platform, but that seemed too basic. I wanted to create a Pokemon performing some attack move.
How about Mewtwo using his signature attack, Shadowball?

Image source

Process I first started by picking out a Mewtwo model from Thingiverse. There were several to choose from, but I selected one at random which happened to be Mewtwo from Super Smash Bros on the WiiU.

I then went to find some type of magic-ball-attack type shape. I searched for "fireball" on Thingiverse and was thrilled to find this fireball file.

I imported both STL models into Rhino and was surprised to find Mewtwo so small, roughly the same size as the fireball. However, I noticed the fireball was extremely intricate as a mesh, so I opted to size it down rather than size up the Mewtwo. I used the "scale" function to decrease the fireball size to some arbitrary amount that would "fit" into Mewtwo's hand.

However, I noticed that the fireball was still extremely intricate with an upwards of 90,000 polygons. For both computing power and possible printing, I wanted to decrease the polygones. I used "ReduceMesh" multiple times, checking the quality of the render each time before processing it again, bringing the count down to around 10,000. I then used "QuadRemesh" which reduced the polygon count by changing the mesh type, but lost a bit of fidelity. I toggled these settings until I found a fidelity and polygon count I was happy with.
Mewtwo was fine so I did not immediately look at reducing his polygons.

Both meshes were already closed when I downloaded them, and after manipulating, they remained closed. I put the fireball into Mewtwo's hand and then joined the two meshes using "MeshBooleanUnion". When placing the fireball, I took care to ensure it was in Mewtwo's hand in a reasonable manner--Mewtwo wouldn't likely be "holding" a Shadowball attack moreso as he would be creating it within his palm, so I tried to place the fireball around his hand accordingly. Unfortunately, that means post-print processing of this model may not be easy.

Once together, I tried manipulating the polygon count of the complete mesh again, using "Reduce Mesh" and "QuadRemesh", as I found that merging meshes ends up processing them back to triangular-polygon meshes. Knowing that "QuadRemesh" loses a bit of fidelity on the mesh, frequently changing the mesh type ends up degrating the model. In the end, I think I like the triangular-polygon mesh best for this model, but I could probably afford to continue reducing the mesh of the fireball, since the most important features are the fidelity of Mewtwo, not the attack.

I also tried fiddling with different features of the model such as changing Mewtwo's tail placement but found the process overall not worth the effort--this might be something better done in Blender or Maya since I'm already dealing with a closed mesh with lots of organic curves.
See my lamp for working with more calculated curves and creation of meshes.
